# `area` Tag Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an optional single-string `area` tag to events so a concept spanning many files ("email-change", "billing") is reliably queryable, independent of file `scope`.

**Architecture:** New `area` column on the `events` table (schema v6, auto-migrated + backfilled). A pure `areas.py` module infers a default area from a `.engram/areas.json` path→area prefix map; explicit `area` always wins. `scope` stays file-paths-only. Query, FTS, and briefing focus-matching become area-aware.

**Tech Stack:** Python 3.12+, SQLite (WAL + FTS5), Click CLI, FastMCP. Dataclasses for models. Only runtime dep is `click`.

## Global Constraints

- Python 3.12+ floor; single runtime dependency (`click`) — no new deps.
- Dataclasses for models (not Pydantic); Click for CLI (one command per function); FastMCP decorators for MCP tools.
- SQLite queries use parameterized statements — never string interpolation of values.
- All store operations use try/finally with `store.close()`.
- Schema is auto-migrated in `store.py`; bump `SCHEMA_VERSION` and add a `version < N` block — never a manual migration step.
- Run the whole suite with `.venv/bin/python -m pytest tests/ -q`. Baseline is 306 passing.
- `.engram/events.db` is machine-local; never commit it. Public repo — no personal paths/emails/secrets in committed files.
- `area` is a single nullable string. One event = one area. `scope` is documented file-paths-only but NOT enforced; existing data is never rewritten.

```python
"""Path->area inference from an optional .engram/areas.json map."""

import json
from dataclasses import dataclass
from pathlib import Path


@dataclass
class AreaRule:
    prefix: str
    area: str


def load_area_map(project_dir: Path) -> list[AreaRule]:
    """Load area rules from <project_dir>/.engram/areas.json.

    Missing file, unreadable file, malformed JSON, or a bad shape all
    return an empty list — inference is best-effort and never fatal.
    """
    path = Path(project_dir) / ".engram" / "areas.json"
    if not path.exists():
        return []
    try:
        data = json.loads(path.read_text())
    except (json.JSONDecodeError, OSError, ValueError):
        return []
    rules_raw = data.get("rules") if isinstance(data, dict) else None
    if not isinstance(rules_raw, list):
        return []
    rules: list[AreaRule] = []
    for r in rules_raw:
        if (isinstance(r, dict)
                and isinstance(r.get("prefix"), str)
                and isinstance(r.get("area"), str)):
            rules.append(AreaRule(prefix=r["prefix"], area=r["area"]))
    return rules


def infer_area(scope: list[str] | None, rules: list[AreaRule]) -> str | None:
    """Infer an area from scope paths using longest-prefix matching.

    For each path in scope order, pick the longest matching prefix. The
    first path that yields a match determines the area. Returns None when
    scope is empty, rules is empty, or nothing matches.
    """
    if not scope or not rules:
        return None
    for path in scope:
        best: AreaRule | None = None
        for rule in rules:
            if path.startswith(rule.prefix):
                if best is None or len(rule.prefix) > len(best.prefix):
                    best = rule
        if best is not None:
            return best.area
    return None
```

```python
def format_event_compact(event: Event) -> str:
    """Single-line compact format for one event."""
    ts = _short_timestamp(event.timestamp)
    scope = _scope_str(event.scope)
    scope_part = f" {scope} —" if scope else " —"
    area_part = f" {{{event.area}}}" if getattr(event, "area", None) else ""
    links = f" (links: {len(event.related_ids)})" if event.related_ids else ""
    priority_tag = f" [{event.priority.upper()}]" if event.priority not in ("normal", None) else ""
    return f"[{ts}] [{event.event_type.value}]{priority_tag} [{event.agent_id}]{scope_part}{area_part} {event.content}{links}"
```

## Notes for the implementer

- **FTS rebuild is the one non-obvious bit.** Adding a column to an FTS5 external-content table means dropping and recreating the virtual table and its insert trigger, then `INSERT INTO events_fts(events_fts) VALUES('rebuild')` to repopulate from `events`. Tasks 2 (fresh schema) and 3 (migration) must keep the FTS column list (`content, scope, area`) and trigger identical.
- **`db_path.parent.parent` is the project dir** because the DB lives at `<project>/.engram/events.db`. The migration and MCP `post_event` both rely on this to find `areas.json`.
- **Area is resolved at write time and frozen.** Editing `areas.json` later does not retroactively re-tag stored events (except the one-time migration backfill). This is intentional and matches how `scope` behaves.
- **Do not enforce that `scope` is file-paths-only.** It stays documentation-only; old events keep whatever they have.
- After all tasks, the package version bump (1.6.1 → 1.7.0) is still the separate release-commit task — not part of this plan.
